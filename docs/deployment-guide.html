Local Setup

Install tools and configure your .env file on your computer

⏱ ~15 mins

📦 Install Required Tools

Check if Python and Git are installed:

python --version
# Should show Python 3.11 or higher

git --version
# Should show git version 2.x

If not installed:

⬇ Download Python 3.11+ ⬇ Download Git

📂 Extract and Set Up Backend

After extracting the portfolio.zip you downloaded:

# Go into backend folder
cd portfolio/backend

# Create a virtual environment (isolates packages)
python -m venv venv

# Activate it — Windows:
venv\Scripts\activate

# Activate it — Mac/Linux:
source venv/bin/activate

# You should see (venv) at start of your terminal line

# Install all packages
pip install -r requirements.txt

# This will take 2-3 minutes (downloading packages)

🔐 Create Your .env File

In the portfolio/backend/ folder, create a file called .env (no extension):

# Windows — in Command Prompt (not PowerShell):
copy .env.example .env
notepad .env

# Mac/Linux:
cp .env.example .env
nano .env

Fill in these values in the .env file:

VariableWhere to get it
MONGO_URLMongoDB Atlas → Connect → Drivers → copy string, replace <password>
DB_NAMEKeep as portfolio
JWT_SECRETAny random string, min 32 chars e.g. myPortfolio2024SecretKey@#$!xyz789
ADMIN_EMAILYour email (you'll use this to log in as admin)
ADMIN_PASSWORDA strong password you'll remember
GROQ_API_KEYFrom console.groq.com → API Keys (starts with gsk_...)
CLOUDINARY_CLOUD_NAMEFrom Cloudinary dashboard
CLOUDINARY_API_KEYFrom Cloudinary dashboard
CLOUDINARY_API_SECRETFrom Cloudinary dashboard

🧪 Test Locally

# Make sure you're in portfolio/backend/ with venv active
uvicorn main:app --reload --port 8000

Open your browser and test these URLs:

http://localhost:8000 → Should show: API is running 🚀
http://localhost:8000/docs → Interactive API docs (Swagger UI)
http://localhost:8000/health → Should show: {"status":"ok"}
💡 Test the frontend: Just open portfolio/frontend/index.html in Chrome/Firefox. The site will load with empty projects (normal — you haven't added any yet).

Push Code to GitHub

Render deploys directly from GitHub — one push = auto deploy

⏱ ~10 mins

1️⃣ Create a New GitHub Repository

  1. Go to github.com/new
  2. Repository name: my-portfolio
  3. Set to Public (required for free Render)
  4. Do NOT check "Add README" or any other options
  5. Click Create repository
  6. Copy the repo URL — looks like https://github.com/YOUR_USERNAME/my-portfolio.git

2️⃣ Initialize Git and Push

From the portfolio/ root folder (not inside backend):

# Initialize git
git init

# Stage all files
git add .

# Create first commit
git commit -m "Initial portfolio commit"

# Connect to your GitHub repo (replace YOUR_USERNAME)
git remote add origin https://github.com/YOUR_USERNAME/my-portfolio.git

# Push to GitHub
git branch -M main
git push -u origin main
⚠ Important: The .gitignore file already excludes your .env file and venv/ folder. Never commit your .env — it contains secrets!

3️⃣ Verify on GitHub

Go to github.com/YOUR_USERNAME/my-portfolio — you should see all your files. Check that .env is NOT listed (good — it's secret).

✅ If you see backend/, frontend/, docs/ folders — you're ready to deploy!

⏱ ~15 mins

  1. Go to dashboard.render.com
  2. Click "New +" → "Web Service"
  3. Click "Connect a repository" → select my-portfolio
  4. Fill in the settings exactly as below:
FieldValue
Nameportfolio-backend
RegionSingapore (closest to India)
Root Directorybackend
EnvironmentPython 3
Build Commandpip install -r requirements.txt
Start Commanduvicorn main:app --host 0.0.0.0 --port $PORT

Scroll down to "Environment Variables" section. Add each one:

KeyValue (your actual values)

https://portfolio-backend-xxxx.onrender.com/

Deploy Frontend

Update the API URL and deploy your portfolio site

⏱ ~10 mins

✏️ Update API URL in frontend/index.html

Open portfolio/frontend/index.html in any text editor. Find this line (around line 335):

: 'https://your-backend.onrender.com'; // ← UPDATE AFTER DEPLOY

Replace it with your actual Render backend URL:

: 'https://portfolio-backend-xxxx.onrender.com';
⚠ Replace xxxx with your actual Render URL — don't use xxxx literally!

📤 Push the Update to GitHub

# From portfolio/ root folder
git add frontend/index.html
git commit -m "Set production API URL"
git push

  2. Click "New +" → "Static Site"
  3. Select your my-portfolio repo
  4. Fill in:
FieldValue

Build Commandleave completely empty
Publish Directory. (just a dot)
  1. Click "Create Static Site"
  2. Deploys in ~1 minute
  3. Your live URL: https://portfolio-frontend-xxxx.onrender.com 🎉
🎉 Your portfolio is LIVE! Open the URL and you should see your full portfolio website with the AI chatbot button in the bottom right.

🔄 Future Updates (Auto Deploy)

Every time you change any code, just run:

git add .
git commit -m "your change description"
git push

Render detects the push and auto-redeploys in ~2 minutes. No manual steps needed!

Fill Your Portfolio

Use the admin panel to add your content — the AI learns from it

⏱ ~20 mins
💡 How it works: Everything you add through the admin panel gets saved to MongoDB AND indexed in ChromaDB. The Groq AI chatbot reads ChromaDB to answer visitor questions about you!

🔐 Step 1 — Log In as Admin

  1. Open your live portfolio site
  2. Scroll to the very bottom of the page
  3. Click the small "🔐 Admin" text in the footer
  4. Enter your ADMIN_EMAIL and ADMIN_PASSWORD from your .env
  5. You'll see an admin bar appear at the bottom of the page

📄 Step 2 — Add Your Resume (Do this FIRST!)

This is the most important step — the chatbot uses your resume to answer questions.

  1. Click "✏ Resume" in the admin bar
  2. Fill in your name, title (e.g. "Big Data Analytics | Automation Engineer")
  3. Write a 3-4 line professional summary about yourself
  4. Add your email, phone, location (Goa, India), LinkedIn, GitHub
  5. Add your skills one by one → type the skill name → press Enter
  6. Upload your PDF resume (optional but recommended)
  7. Click Save Resume — your homepage updates instantly!
Tips for your summary: Mention your PGDM in Big Data Analytics, your Automation Engineer and Data Science Engineer roles, your dashboard experience, and DevOps skills. More detail = better chatbot answers!

🚀 Step 3 — Add Your Projects

  1. Click "+ Project" in the admin bar
  2. Fill in Title, Short Description (1-2 lines), Tech Stack (press Enter after each)
  3. Add GitHub URL and/or Live Demo URL if available
  4. Check "Featured" for your best 1-2 projects
  5. Click Save Project
  6. Then click Edit on the project → upload a screenshot image
💡 Project ideas to add: Dashboard projects, automation scripts, ML models, data pipelines, any GitHub projects you've worked on!

🏆 Step 4 — Add Your Certifications

  1. Click "+ Certification" in the admin bar
  2. Fill in the certificate name, issuer (e.g. Coursera, Google, AWS, etc.)
  3. Add the issue date in YYYY-MM format (e.g. 2024-03)
  4. Paste the credential URL (from LinkedIn or the certificate email)
  5. Add skills covered by this cert (press Enter after each)
  6. Click Save Certification

🤖 Step 5 — Test the AI Chatbot

  1. Click the blue chat button (bottom right corner)
  2. Try asking these questions:
    • "What are your skills?"
    • "Tell me about your projects"
    • "What certifications do you have?"
    • "What is your educational background?"
    • "How can I contact you?"
  3. The AI should answer using YOUR actual resume and projects!
✅ If the chatbot gives accurate answers about you → everything is working perfectly!

Troubleshooting

Common issues and exactly how to fix them

Reference

⚠️ Common Issues

❌ Backend build fails on Render ▼
Check these in order:

1. Go to Render → your service → Logs tab → scroll to the red error
2. Most common: wrong Python version. Make sure Render uses Python 3.11
3. Go to Render → Environment → add: PYTHON_VERSION = 3.11.0
4. Also check that Root Directory is set to backend (not portfolio/backend)
❌ "MongoDB connection failed" in logs ▼
Two things to check:

1. IP Whitelist — Go to MongoDB Atlas → Network Access → Make sure 0.0.0.0/0 is listed. If not, add it.

2. Connection string — In Render env vars, check your MONGO_URL. Make sure:
• You replaced <password> with your actual password
• No extra spaces before or after the URL
• The database name is in the URL before the ?: .../portfolio?retry...
❌ Site loads but shows "Couldn't load projects" ▼
The frontend can't reach the backend. Check:

1. Open browser DevTools (F12) → Console tab → look for CORS or network error
2. In frontend/index.html, verify the API URL is your actual Render backend URL (not the placeholder)
3. Check the backend is running: visit https://your-backend.onrender.com/health
4. Free Render services sleep after 15 mins — first request can take 30-60 seconds to wake up. Wait and refresh!
❌ Admin login says "Invalid credentials" ▼
The email/password don't match what's in your Render env vars.

1. Go to Render → your backend service → Environment
2. Check ADMIN_EMAIL and ADMIN_PASSWORD values exactly
3. Make sure you're typing them exactly the same in the login form (case-sensitive!)
4. After changing env vars in Render → click "Manual Deploy" to restart
❌ Chatbot says "Chat error" or doesn't respond ▼
Groq API issue:

1. Check GROQ_API_KEY in Render env vars — must start with gsk_
2. Go to console.groq.com → verify the key is active
3. Check Render logs for the specific error message
4. If it says "rate limit" — wait 1 minute and try again (free tier has limits but they reset quickly)
❌ Image upload fails ▼
Cloudinary configuration issue:

1. Double-check all 3 Cloudinary values in Render: CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET
2. Go to Cloudinary dashboard → Settings → Upload → make sure uploads are enabled
3. Try uploading a smaller image (under 5MB)
4. Check Render logs for the Cloudinary error message
⚡ Backend takes 30-60 seconds to respond (first request) ▼
This is normal for Render's free tier! Free services "sleep" after 15 minutes of no traffic.

Options:
1. Free fix: Use UptimeRobot.com (free) to ping your backend URL every 14 minutes → it never sleeps!
2. Paid fix: Upgrade to Render Starter ($7/month) → always on, no sleep

UptimeRobot setup: Sign up free → New Monitor → HTTP → paste your backend URL → every 14 minutes
❌ "git push" asks for username/password ▼
GitHub no longer accepts passwords — you need a Personal Access Token:

1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token → check repo scope → Generate
3. Copy the token (starts with ghp_...)
4. When git asks for password → paste the token instead

Or easier: install GitHub CLI and run gh auth login
